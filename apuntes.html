se debe tener instalado node json para ver version ejecutar node -v v8.9.4 Para empezar debemos crear un archivo package.json inicial rápido: npm init llenar los campos solicitados Ahora tenemos que instalar el postcss-cli en el entorno de trabajo y no global. npm install postcss-cli --save-dev con esto ya se agrego al package.json "postcss-cli": "^6.1.2" ademas se creo la carpeta node_modules Para ejecutar el comando en el entorno de trabajo ejecutamos: npx postcss-cli --version (*) Documentación PostCSS CLI: https://github.com/postcss/postcss-cli 1. Para transformar un archivo en ser ejecuta el siguiente comando: relacionar el CSS que vamos a desarrollar con el de producción npx postcss src/css/home.css -o dist/css/home.css Cambiar la ruta en nuestro index.html al archivo transformado. lo que tenemos en src es para desarrollo y lo que tenemos en dist es para producción 2. Transformar un ves que se realice un cambio: npx postcss src/css/home.css -o dist/css/home.css -w Esto permite que si actulizamos el css de desarrollo se actulice el de producción 3. Uso de Plugins de PostCSS: npx postcss src/css/home.css -o dist/css/home.css -w -u [ -u | --use] uso de plugins de PostCSS para ver que plugins existen e instalarlo www.postcss.parts utilizando el plugin autoprefixer, coloca los prefijos para que funcione correctamente en los diferentes navegadores instalando autoprefixer npm install --save autoprefixer para correrlos de forma manual ejecutamos npx postcss src/css/home.css -o dist/css/home.css -w -u autoprefixer www.caniuse.com esta pagina sirve para ver las capacidades de los navegadores creamos un archivo con el nombre postcss.config.js en la raiz module.exports = { plugins:[ require('autoprefixer')({ grid:true }) ] } se puede prender y apagar en un area especifica /*autoprefixer: off*/
